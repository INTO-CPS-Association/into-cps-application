Traceability Hints and Tips

The INTO-CPS tool chain (including the INTO-CPS Application, Overture, 20-sim, OpenModelica, Modelio and analyses) records traceability and provenance data when modelling, simulating and analysing. This data is stored in a database which runs whilst the INTO-CPS application is open.

The Neo4J viewer allows users to view all recorded traceability data for a project and also to query this data for uses such as impact analysis.

Installing Neo4j

The INTO-CPS Application automatically records traceability and provenance links, however, to view the database of links, users must install the Neo4j viewer. This is included in the lastest release of INTO-CPS tools (select Show Download Manager and download Neo4J Community Edition - External Link

Once downloaded, extract the folder to the install or install-downloads folder.

Querying the Traceability database

Once logged in, users are able to interact with the database. Neo4j uses the Cypher language to query the database. To enter new queries, use the textbox at the top of the view. For example, to see the full traceability graph enter:

MATCH (nodes) RETURN nodes;

For more details on queries, see Cypher manual.

Explaining the view

A traceabilty graph comprises a collection of nodes and edges. Nodes comprise entities, activites and agents. For more information, see Appendix A of INTO-CPS Deliverable D3.1a.
